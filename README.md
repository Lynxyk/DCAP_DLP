# DCAP DLP
**DCAP DLP** - система защиты данных (Data Loss Prevention) с десктопным JavaFX-клиентом и Spring Boot REST-бэкендом.  
Проект реализует проверку текстов и файлов на наличие запрещённой информации, ведёт аудит действий пользователей и управляет политиками безопасности.
## Основные возможности
- Регистрация и авторизация пользователей с хэшированием паролей (**BCrypt**)
- Проверка содержимого файлов на стоп-слова, регулярные выражения и запрещённые расширения
- Извлечение текста из документов и изображений с помощью **Apache Tika + Tesseract OCR**
- Система ролей (администратор, пользователь) с различными политиками безопасности
- Логирование действий пользователей в таблицу **LogDsap**
- Сброс и подтверждение пароля через email
- Админ-панель для управления пользователями, стоп-словами, регулярками и ролями
- Интеграция клиента и сервера через REST API
## Архитектура проекта
Многослойная структура:
- **Frontend (JavaFX)** графический интерфейс, контроллеры, утилиты (`TikaTextExtractor`, `TextCheckService`, `AuthService`)
- **Backend (Spring Boot)** REST-контроллеры (`TextCheckController`, `UserController` и др.), сервисы и репозитории
- **Бизнес-логика** проверка содержимого по словарям и шаблонам с учётом ролей пользователей
- **Хранилище данных** PostgreSQL, Spring Data JPA (Hibernate)
- **Безопасность** Spring Security, BCrypt-хэширование, email-верификация
## Принцип работы
1. Пользователь авторизуется в JavaFX-клиенте, данные проверяются через `/users/check-credentials`
2. Выбирает файл для анализа `TikaTextExtractor` извлекает текст с помощью **Apache Tika** и **Tesseract OCR**
3. Клиент отправляет текст и информацию о файле на эндпоинт `/check/text`
4. Сервер анализирует содержимое по стоп-словам, регуляркам и запрещённым расширениям.
5. Результат проверки возвращается пользователю, событие сохраняется в `LogDsap`
6. Администратор может просматривать логи и редактировать политики безопасности
